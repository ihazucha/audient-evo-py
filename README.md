# Audient EVO 4 - reverse engineered multiplatform controller

## EVO 4 Specification

Functions:
- input select
  - gain knob
  - mute toggle
  - phantom power toggle
- monitor mix
  - input/output ratio knob
- output volume
  - volume knob
  - mute toggle

HW:
- USB:
  - VID: 0x2708
  - PID: 0x0006

SW (Custom driver + App for control):
- Windows/MacOS only

## Installation

Requirements:
- pyusb: https://github.com/pyusb/pyusb
  - libusb (backend): https://github.com/libusb/libusb

## USBPcap
- Host -> Dev (SET CUR) - generated by changes in app:
  usb.bmRequestType == 0x21
- Host -> Dev (GET CUR/MEM) - device state polling:
  usb.bmRequestType == 0xA1

## Dev Workflow

1. On Windows:
  - run EVO 4 App
  - capture comms using USBPcap, filter in Wireshark
  - map requests to functions (volume, mute, ...)
2. Update `audient.py`, implement functionality, test.
  - the device sends info to host about the value set by a request
  - `audient.py` should also implement a poll (GET) requests to a function
